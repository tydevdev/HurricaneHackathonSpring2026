# Slopternet

## One-Sentence Pitch

Slopternet is a website that looks instantly, obviously AI-generated from the first screen, then collapses into an even more haunted and unstable web artifact as the visitor keeps clicking, searching, accepting, optimizing, and trying to understand it.

## Core Idea

This is the louder variation of the concept. It should look like AI slop immediately: glossy, overconfident, generic, repetitive, and weirdly hollow. The visitor should know right away that the page was made by systems trying to imitate a website without understanding why the website exists.

It is a website first and a game second. It should not feel like a traditional game with levels, enemies, or a visible win condition. It should feel like a low-care AI-generated marketing site that becomes interactive evidence for its own warning: if the web is filled with automated pages optimized for volume instead of care, truth, craft, and maintainability, the internet itself starts to feel structurally unsafe.

The visitor plays by behaving like a normal internet user. They click buttons, open menus, accept cookies, ask the chatbot questions, search for information, refresh content, and try to fix obvious problems. Each interaction makes the site degrade a little more. The page becomes the antagonist.

Sibling variation: `quiet-collapse/` explores the same warning through a normal-looking site that slowly becomes creepy and broken. `slopternet/` should stay more immediate, satirical, and visibly AI-sloppy from the start.

## Inspiration Notes

Welcome Home is useful as a structural reference because it is an interactive website with a friendly surface, hidden unease, ordinary page elements that reward close inspection, and a story that emerges through exploration instead of exposition.

Reference anchors:

- Official site: <https://welcomehomerestorationproject.net/>
- Overview: <https://knowyourmeme.com/memes/subcultures/welcome-home-arg>

Borrow the pattern, not the specifics:

- Friendly surface that becomes suspicious.
- Secrets embedded in normal interface pieces.
- Hidden pages, strange artifacts, and corrupted media.
- Exploration as the main form of play.
- A tonal slide from colorful and harmless to wrong, watched, and unstable.

Avoid copying the actual creative identity:

- No puppets.
- No neighborhood.
- No restoration-project framing.
- No copied characters, names, visual motifs, or lore.
- No direct imitation of the Welcome Home art style.

## Message

Slopternet should make the visitor feel this argument rather than read it as a lecture:

The future internet becomes fragile when automated systems can generate endless sites, but nobody is responsible for whether those sites are true, stable, maintained, accessible, or human.

The ending should land on a clear warning, but the experience should earn that warning through collapse.

## Experience Arc

### Stage 1: Instant Slop

The site opens as a painfully generic AI productivity or growth platform landing page. It should look like the output of a prompt that asked for "a modern SaaS landing page with premium vibes" and then stopped caring.

Possible surface elements:

- Hero headline about unlocking smarter workflows with future-ready intelligent synergy.
- Generic dashboard mockup.
- Vague customer logos.
- Feature cards with meaningless promises and duplicated structure.
- Testimonials that sound polished, fake, and suspiciously interchangeable.
- Buttons like `Generate Better Copy`, `Optimize Layout`, `Refresh Content`, and `Start Free`.
- Stock-like generated art that almost matches the product but not quite.
- Trust badges that do not say who is trusting whom.

The page should be usable and stable at first, but not tasteful. The joke is that it is already bad before it starts breaking.

### Stage 2: Small Wrongness

Interactions begin to produce slightly broken results.

Examples:

- A button changes its label to a nearby but wrong CTA.
- Testimonials repeat one phrase too many times.
- Layout spacing drifts by a few pixels after each click.
- Icons mismatch their labels.
- The chatbot answers confidently but does not answer the question.
- Search results include pages that do not exist yet.
- A cookie banner returns after being accepted.

The visitor should still feel in control, but the page has started lying more aggressively.

### Stage 3: Structural Decay

The site becomes visibly unstable.

Examples:

- Cards duplicate themselves.
- Navigation links lead to the wrong sections.
- Images fail and reveal prompt fragments in their alt text.
- Modals stack behind each other.
- The testimonial carousel creates fake people faster than it can display them.
- Contrast worsens.
- Buttons slide away from the cursor.
- Copy contradicts itself between sections.
- The footer grows with autogenerated policy pages no one can open.

At this stage, the page should feel game-like: the visitor is poking the site to see what breaks next.

### Stage 4: The Machinery Shows

The page begins revealing hidden material from inside the generation pipeline.

Possible artifacts:

- Broken prompt fragments.
- Placeholder instructions like `[INSERT TRUST SIGNAL HERE]`.
- Fake analytics events.
- Moderation notes that do not match visible content.
- Version histories that imply no human reviewed the page.
- Hidden messages in repeated phrases.
- Dead internal links with page titles that sound ominous.
- A changelog that keeps rewriting itself.

This is where the Welcome Home-style influence matters most: ordinary web objects become clues, but the tone remains rooted in corrupted AI slop rather than hidden lost-media nostalgia.

### Stage 5: Almost Unusable

The page is barely stable, but not random. It should collapse with intention.

Possible end-state behaviors:

- The layout tears apart but leaves one readable column.
- The chatbot admits it cannot tell whether the site is true.
- The search box only returns warnings.
- The `Fix Website` button makes everything worse.
- The page title keeps changing.
- Navigation becomes a list of broken promises.
- A final message appears only after enough interactions:

> The web does not break all at once. It breaks every time nobody checks what was generated.

The final message should be readable, direct, and visually protected from the surrounding collapse.

## Core Interactions

The site should reward curiosity. Every ordinary interaction should have a first-click behavior and a later corrupted behavior.

Candidate controls:

- `Generate Better Copy`: rewrites visible text into increasingly repetitive nonsense.
- `Optimize Layout`: improves one section while damaging another.
- `Accept Cookies`: closes the banner, then returns with more invasive language.
- `Refresh Content`: replaces useful text with fresher but less meaningful text.
- Fake chatbot: starts helpful, then loops, contradicts itself, and leaks prompt fragments.
- Broken search: finds strange hidden pages and impossible documentation.
- Testimonial carousel: creates fake customers, duplicated names, and suspiciously perfect quotes.
- `Fix Website`: presented as a rescue button, but it accelerates collapse.
- Navigation menu: starts normal, then routes to incorrect or newly corrupted sections.
- Footer links: multiply into fake legal, trust, and policy pages.

## Game-Like Shape

The visitor should be able to discover more by interacting, but the experience should still feel like browsing a website.

Possible progression model:

- Track a hidden `instability` score.
- Every interaction increases instability.
- Some interactions unlock specific artifacts at thresholds.
- Repeated clicks on the same element reveal deeper corruption.
- Optional hidden links appear only after the page has degraded enough.
- The final warning appears after a set number of discoveries, not on a timer.

There should be no score label in the UI. The visitor should infer progress from the site falling apart.

## Visual Direction

The opening look should be intentionally and visibly AI-generic:

- Clean SaaS layout.
- Overused gradients.
- Slightly sterile cards.
- Polished but empty copy.
- Smooth spacing.
- Fake trust marks.
- Bland dashboard preview.
- Unconvincing generated product imagery.
- Repeated icon metaphors.
- Headlines that sound machine-assembled.

The degradation should become more physical and unsettling:

- Layout tearing.
- Repeated component ghosts.
- Flicker.
- CSS drift.
- Misaligned grids.
- Stacking popups.
- Broken image frames.
- Repeated slogans.
- Hallucinated icons.
- Cursor avoidance.
- Sections that appear generated from the wrong brief.
- Text that seems to remember earlier clicks.

The site should not become visually random. It should look like a system failing in stages.

## Tone

Start:

- Corporate.
- Vague.
- Optimistic.
- Over-polished.
- Hollow.
- Already suspiciously generated.

Middle:

- Confident but incorrect.
- Glitchy.
- Over-personalized.
- Slightly invasive.

End:

- Haunted.
- Brittle.
- Direct.
- Still darkly funny in places.

## Future Build Approach

When this becomes a real subproject, build it as an independent Vite React app inside `slopternet/`.

Likely structure:

- React state tracks instability, discoveries, repeated interactions, and visible corruption.
- CSS variables shift with instability to create controlled layout drift.
- Components receive corruption states instead of relying on random one-off DOM hacks.
- Hidden artifacts unlock at specific thresholds.
- Local storage can remember whether the visitor has already seen the final collapse.
- A reduced-motion mode keeps the concept accessible without flicker-heavy effects.
- Mobile layout should degrade differently but remain intentionally playable.

Root hub integration should wait until the app has a committed `dist/` build.

## Verification Goals For The Future Build

Before calling the future website done:

- Verify desktop and phone layouts.
- Verify touch and mouse interaction paths.
- Verify the degradation remains usable long enough to discover the message.
- Verify reduced-motion behavior.
- Verify the final warning is readable.
- Verify the root hub links only after `dist/` exists.
